# Product Recommendation Engine - Collaborative Filtering & Matrix Factorization

> Building a personalized recommendation system from raw user behavior data.  
> Popularity Baseline · Item-CF · SVD Matrix Factorization · Business Impact Analysis

---

## Overview

This project builds and evaluates three recommendation approaches on 2.76 million user behavior events from a real e-commerce platform. The same techniques apply directly to SaaS products — replacing "items" with features, plans, or content, and "purchase events" with upgrades or activations.

The project goes beyond model metrics: every result is translated into a **revenue impact estimate** so stakeholders can make informed decisions about which approach to deploy.

## Key Results

| Model | Hit Rate @ 10 | vs Baseline |
|-------|--------------|-------------|
| Popularity baseline | 2.4% | — |
| SVD Matrix Factorization | 14.2% | +492% |
| **Item-based CF** | **47.2%** | **+1,867%** |

**Hit Rate @ 10:** probability that the user's next purchase appears in the top-10 recommendations (leave-one-out evaluation on 500 users).

## Methodology

### Implicit Feedback Weighting
User actions are weighted by purchase intent — the same approach used by Spotify (streams vs saves) and Netflix (completion rate vs explicit ratings):

| Event | Weight | Rationale |
|-------|--------|-----------|
| view | 1 | Passive interest |
| addtocart | 3 | Strong intent signal |
| transaction | 5 | Confirmed purchase |

### Model 1 — Popularity Baseline
Recommend the globally most purchased items to every user. No personalization. Sets the floor for any ML approach to beat.

### Model 2 — Item-based Collaborative Filtering
Compute cosine similarity between items in the user-item interaction space. For each user, find items similar to their interaction history and score them by accumulated similarity. Item-CF is more stable than user-CF because item catalogs are more static than user preferences.

### Model 3 — SVD Matrix Factorization
Decompose the sparse user-item matrix into 50 latent dimensions using Truncated SVD. Each user and item is represented as a dense vector; recommendations are generated by dot product scoring. Conceptually equivalent to Netflix's collaborative filtering.

### Evaluation — Hit Rate @ K (leave-one-out)
For each evaluation user: hold out their most recent transaction, generate top-K recommendations from the remaining history, check if the held-out item appears in the list. This directly answers: *"Would our system have recommended what the user actually bought?"*
